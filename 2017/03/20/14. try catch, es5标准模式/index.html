try..catch es5标准模式

2017/03/20

本文于364天之前发表，文中内容可能已经过时。

try..catch

  • 举个例子理解
  • 如果try里面有多个错误怎么办呢？

    —->如果try里面第一个错误是在第三行，那么系统执行到第三行的时候直接跳进catch里，不会看第三行以下的错误代码。
  • 错误类型
    • EvalError: eval()使用与定义不一致;
    • RangeError: 数值越界
    • ReferenceError: 非法或不能识别的引用数值 (引用错误，赋值错误 例如：当一个变量没经声明就使用，函数名声明就使用，例如 aa is not defined)
    • SyntaxError: 发生语法解析错误 (低级错误)
    • TypeError: 操作数类型错误
    • URLError: URL处理函数使用不当 (引用地址错误)

ES5标准模式

  • 用

var obj = {
name : “obj”
}
var name = ‘window’;

function test() {
var age = 123;
var name = ‘aa’
with(obj) {
console.log(name);
console.log(age);
}
}
会打印obj 123

作用

  1. 防止污染变量
    with(liuzhaoxu.js) {
    …..里面的代码直接指向liuzhaoxu.js 不会污染全局变量
    }
  2. 简化代码
    with(document) {
    write(‘a’); 这样就不用写document了
    }

但是一般不用，了解就行，因为他改变了作用域链；导致系统编译缓慢

function Test() {
console.log(this);
}
Test();
严格模式下 返回undefined；
非严格模式下 返回window；

function test(name, name) {
console.log(name);
}
test(1,2)
重复的参数在es3里不会报错 返回2
但是es5是报错的
